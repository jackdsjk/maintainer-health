# maintainer-health v0.4.0

Feature release focused on making audit output more actionable for real
maintenance work.

## Highlights

- Adds category, priority, and effort metadata to every check.
- Sorts failed checks by maintainer impact instead of file order.
- Upgrades `--fix-plan` with:
  - Top three recommended fixes.
  - Category-grouped full checklist.
  - Finding, next action, priority, and effort for each failed check.
- Expands Markdown reports with category, priority, and effort columns.
- Updates package version metadata to `0.4.0`.

## Why This Matters

Maintainers usually do not need a longer list. They need to know what to fix first.
This release makes `maintainer-health` more useful during a cleanup session, CI
failure, or pre-release review.

## Validation

- `.venv/bin/python -m ruff check .`
- `.venv/bin/python -m pytest`
- `.venv/bin/python -m maintainer_health /tmp --fix-plan`
- `.venv/bin/python -m maintainer_health . --profile release-ready --fail-under 90`
