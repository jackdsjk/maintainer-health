# maintainer-health v0.3.0

Feature release focused on making audits match real maintainer jobs instead of a
single generic checklist.

## Highlights

- Adds scenario-based audit profiles:
  - `contributor-ready`
  - `release-ready`
  - `security-baseline`
- Adds `--list-profiles` for profile discovery.
- Includes profile metadata in text, JSON, Markdown, and fix-plan outputs.
- Updates the roadmap around concrete maintainer workflows and product value.
- Updates package version metadata to `0.3.0`.

## Validation

- `.venv/bin/python -m ruff check .`
- `.venv/bin/python -m pytest`
- `.venv/bin/python -m maintainer_health . --profile release-ready --fail-under 90`

## Notes

This release keeps the tool offline-first. The new profiles select different local
checks; they do not add network calls or credential handling.
